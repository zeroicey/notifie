# Notifie 多平台通知系统

## 1. 项目概述

Notifie 是一个轻量级的多平台通知系统，用于在桌面端（Windows、macOS、Linux）显示系统原生通知。支持通过服务器推送，实时将消息广播给所有在线客户端。

## 2. 需求总结

| 维度 | 要求 |
|------|------|
| 场景 | 个人/团队工具（监控告警、CI/CD 状态等） |
| 平台 | 桌面端 (Windows、macOS、Linux) |
| 客户端 | 独立 App，常驻系统托盘 |
| 消息格式 | 纯文本 title + content |
| 显示方式 | 系统原生通知 |
| 服务端功能 | 简单转发，无复杂逻辑 |
| 使用规模 | 小型 (1-10 人团队) |
| 服务器地址 | 用户可配置 |
| 托盘菜单 | 状态 + 设置 + 退出 |

## 3. 技术选型

| 层级 | 技术选择 |
|------|----------|
| 客户端框架 | Tauri 2.x |
| 前端运行时 | Bun |
| 前端框架 | React 19 |
| 后端框架 | Go + Fiber |
| 通信协议 | WebSocket |
| 通知方式 | 系统原生通知 (Tauri Notification API) |

### 技术选型理由

- **Tauri**: 比 Electron 体积小 10-50 倍，性能更好，内存占用更低
- **Bun**: 比 Node.js 更快，开发体验更好
- **WebSocket**: 满足实时通知需求，协议简单
- **Go + Fiber**: 高性能，goroutine 并发模型适合长连接场景

## 5. 数据格式

### 5.1 WebSocket 消息

```json
// 服务器 -> 客户端 (推送通知)
{
  "type": "notify",
  "title": "告警",
  "content": "CPU 使用率超过 90%",
  "timestamp": 1700000000
}
```

### 5.2 HTTP API

```
POST /api/notify
Content-Type: application/json

{
  "title": "告警",
  "content": "CPU 使用率超过 90%"
}
```

响应：
```json
{
  "success": true,
  "message": "Notification sent to X clients"
}
```

## 6. 客户端功能

- [ ] 启动时自动连接 WebSocket 服务器
- [ ] 连接状态显示（托盘图标颜色）
- [ ] 接收消息后调用系统原生通知
- [ ] 可配置服务器地址（通过设置页面）
- [ ] 托盘菜单：状态、设置、退出
- [ ] 设置页面：服务器地址配置

## 7. 服务端功能

- [ ] WebSocket 服务器，接收客户端连接
- [ ] HTTP API，接收外部通知请求并广播给所有客户端
- [ ] 支持 CORS（开发环境）
- [ ] 日志输出

## 9. 部署

### 服务端

```bash
cd apps/notifie-server
go run main.go
# 默认监听 :8080
```

### 客户端

```bash
cd apps/notifie-client
bun run tauri dev    # 开发模式
bun run tauri build  # 构建
```
